## 2. Fullscreen fix

Two controls existed: Mol*'s built-in browser-fullscreen icon (unreliable, the
"weird"-looking one) and the app's own CSS `molFull` button (worked, but never
told Mol* to resize its canvas).

- `frontend/src/components/MolPlayfield.jsx`
  - `PluginConfig.Viewport.ShowToggleFullscreen` set to **false** (removes the
    broken built-in icon).
  - `componentDidUpdate` now calls `this.plugin.handleResize()` (next animation
    frame) whenever the new `fullscreen` prop flips, so the canvas fills the box.
- `frontend/src/App.jsx`
  - The app fullscreen button now renders a clean inline-SVG maximize/minimize
    icon (no more tofu glyph), and its `setState` callback also dispatches a
    `window.resize` event (belt-and-suspenders for Mol*).
  - Passes `fullscreen: st2.molFull` into `MolPlayfield`.

Please QA visually: enter/exit fullscreen and confirm the 3D actually fills the
screen and the icon looks right.

## 6. Pedagogy — what exists vs what was added

The 5.5pro review was written against an older zip. The current build **already
implements** most of its P1 backlog:

- All five interactive lens scenes exist (`renderScene` → `sceneCoev`, `sceneTri`,
  `sceneIpa`, `sceneFape`, `sceneRec`). `sceneCoev` is the covariance →
  inverse-covariance (precision) lab the review asked for (§6.3 MSA/Potts);
  `sceneIpa` is the rotate/translate invariance scene (§6.6); `sceneFape` is the
  mirror/chirality scene (§6.7); `sceneRec` is the recycling fixed-point scene.
- `truthLabels`, a low-confidence lesson panel, and per-lens `boundary`/`q`
  framing already exist; `paperGrounding.js`/`sceneSpecs.js` carry tensor shapes,
  prerequisites and misconceptions (the curriculum graph from §5.3/§6.4).

**Added this session:** a consolidated **"Myth → Reality"** panel on the Library
page (review §11 / §6.1 "recycling is not folding", "pLDDT is not free energy",
"samples are not an ensemble", etc.) — previously the misconceptions were only in
data and scattered per-lens. The new proteins also directly seed lessons:
calmodulin (§6.8 pLDDT-vs-PAE), Ras (cofactor omission), alpha-synuclein
(disorder → low pLDDT), villin (Levinthal / recycling-is-not-folding).

**Still open (future):** dedicated "Protein Basics" onboarding path (§6.2);
AF3 token/diffusion mode (§6.9, P2); PAE click-to-highlight wired into the Mol*
viewer; saved progress/quizzes. These are larger and best done as separate,
build-verified passes.
